# Contributing

This project exists so that solo founders and small teams can get from "vibe-coded prototype" to safe, reliable, secure, monetizable product faster. If you improve the shared library, I'd love for you to contribute those changes back so everyone benefits.

Because the core library is licensed under the Mozilla Public License 2.0 (MPL-2.0), any modified versions of licensed files that you distribute must be made available under the same license. Opening a pull request here is the easiest way to satisfy that requirement.

## What counts as a contribution?

Examples of valuable contributions include:

- Clarifying or expanding existing sections in the core markdown files
- Adding new steps, checklists, or examples that are generally useful to other founders
- Fixing mistakes, broken links, or confusing wording
- Improving security, reliability, or monetization guidance that's broadly applicable

Anything that's specific to your company, product, or private processes can (and should) live in your own `our-context/` directory, which is `.gitignored` and not part of this shared library.

## How to contribute

1. Fork this repository
2. Make your changes to the relevant files in the core library (not in `our-context/`)
3. Include a short description of what you changed and why in the pull request
4. If your change is based on a real-world incident or success story, feel free to include a brief, anonymized note so others can learn from it too

By submitting a contribution, you agree that your changes to the core library files are licensed under MPL-2.0, the same license as the rest of the project.

If you're unsure whether something belongs in a contribution (shared library) or in your private `our-context/` directory, open an issue or start a discussion and we can sort it out together.
